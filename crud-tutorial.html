Full-Stack Web Development Crash Course: Building a CRUD App

Welcome to this crash course on full-stack web development! In the next 45 minutes, we'll learn how to build a simple CRUD application using modern web technologies, and leverage GitHub Copilot to accelerate your development process.

Step 1: Understanding CRUD and Web Architecture

What is CRUD?

  • Create: Adding new data to your database
  • Read: Fetching and displaying existing data
  • Update: Modifying existing data
  • Delete: Removing data from your database

Front-End vs Back-End

  • Front-End: What users see and interact with (HTML, CSS, JavaScript)
  • Back-End: Server-side code that processes requests and works with databases (Node.js, Express)
  • Database: Where your data is stored and managed (we'll use a simple JSON file for this tutorial, but in production you might use MongoDB, PostgreSQL, or MySQL)
  • Full-Stack: Combining both front-end and back-end development

Step 3: Building Your First Server

ðŸ“šLearn: A server is a computer program that receives requests from users' browsers, processes them, and sends back responses. It's like a restaurant waiter taking orders and bringing food.

Create a file named app.js with the following code:

const express = require('express');
const path = require('path');
const JSONdb = require('simple-json-db');
const app = express();
const db = new JSONdb('./data.json');

// Set up middleware
app.use(express.urlencoded({ extended: true }));
app.use(express.json());

// Set up the view engine
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'ejs');

// Define your routes here
app.get('/', (req, res) => {
  const items = db.get('items') || [];
  res.render('index', { items: items });
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

Step 5: Implementing CRUD Operations

ðŸ“šLearn: Routes tell your server which code to run when someone visits different URLs, like telling a receptionist which department should handle different types of calls.

Add these routes to your app.js:

// Create operation
app.post('/item', (req, res) => {
  const items = db.get('items') || [];
  items.push({
    name: req.body.name,
    description: req.body.description
  });
  db.set('items', items);
  res.redirect('/');
});

// Read operation (already implemented in the main route)

// Update operation - form
app.get('/item/:id/edit', (req, res) => {
  const items = db.get('items') || [];
  const item = items[req.params.id];
  res.render('edit', { item: item, id: req.params.id });
});

// Update operation - process
app.post('/item/:id/update', (req, res) => {
  const items = db.get('items') || [];
  items[req.params.id] = {
    name: req.body.name,
    description: req.body.description
  };
  db.set('items', items);
  res.redirect('/');
});

// Delete operation
app.post('/item/:id/delete', (req, res) => {
  const items = db.get('items') || [];
  items.splice(req.params.id, 1);
  db.set('items', items);
  res.redirect('/');
});

Step 6: Using GitHub Copilot

Setting up GitHub Copilot:

  1. Install GitHub Copilot extension in your code editor
  2. Sign in with your GitHub account
  3. Ensure your account has access to Copilot

Using Copilot for Development:

  • Prompt completions: Type a comment describing what you want to do, and Copilot will suggest code
  • Contextual awareness: Copilot understands the structure of your project
  • Learning from patterns: As you write more code, Copilot adapts to your style

Example Prompts for Copilot:

  • "// Create a function to validate form input"
  • "// Add a route to handle API requests"
  • "// Create a database query to filter items"
