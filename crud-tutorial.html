Full-Stack Web Development Crash Course: Building a CRUD App

Welcome to this crash course on full-stack web development! In the next 45 minutes, we'll learn how to build a simple CRUD application using modern web technologies, and leverage GitHub Copilot to accelerate your development process.

Learning Objectives

  • ✅ Understand the basics of CRUD operations
  • ✅ Set up a Node.js and Express server
  • ✅ Create a simple front-end with HTML/CSS
  • ✅ Connect front-end and back-end using Express and EJS
  • ✅ Use GitHub Copilot to enhance development
  • ✅ Deploy your application

Step 1: Understanding CRUD and Web Architecture

What is CRUD?

  • Create: Adding new data to your database
  • Read: Fetching and displaying existing data
  • Update: Modifying existing data
  • Delete: Removing data from your database

Front-End vs Back-End

  • Front-End: What users see and interact with (HTML, CSS, JavaScript)
  • Back-End: Server-side code that processes requests and works with databases (Node.js, Express)
  • Database: Where your data is stored and managed (we'll use a simple JSON file for this tutorial, but in production you might use MongoDB, PostgreSQL, or MySQL)
  • Full-Stack: Combining both front-end and back-end development

Step 2: Setting Up Your Development Environment

📚Learn: A development environment is where you write and test your code before sharing it with others. Think of it as your digital workshop.

  1. Install Node.js from nodejs.org
  2. Create a new project folder: 
    mkdir my-crud-app
cd my-crud-app
  3. Initialize your project: 
    npm init -y
  4. Install necessary packages: 
    npm install express ejs simple-json-db

Step 3: Building Your First Server

📚Learn: A server is a computer program that receives requests from users' browsers, processes them, and sends back responses. It's like a restaurant waiter taking orders and bringing food.

Create a file named app.js with the following code:

const express = require('express');
const path = require('path');
const JSONdb = require('simple-json-db');
const app = express();
const db = new JSONdb('./data.json');

// Set up middleware
app.use(express.urlencoded({ extended: true }));
app.use(express.json());

// Set up the view engine
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'ejs');

// Define your routes here
app.get('/', (req, res) => {
  const items = db.get('items') || [];
  res.render('index', { items: items });
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

Step 4: Creating Your Views

📚Learn: Views are the set of front-end html pages that will be displaying your application's functionality and data to the user. The "face" and UI of your app.

Tip: Install the "EJS language support" extension in VS Code for syntax highlighting and better editing experience with EJS templates. You can find it by searching for "EJS" in the Extensions panel (Ctrl+Shift+X / Cmd+Shift+X).

  1. Create a folder named views
  2. Inside the views folder, create these files:
    • index.ejs - Main page with item list and create form
    • edit.ejs - Form for editing existing items

index.ejs - Main View

This view handles both displaying items and creating new ones:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>CRUD App</title>

</head>
<body>
    <div class="container mt-5">
        <h1>Item Manager</h1>
        
        <!-- Create Form -->
        <div class="card mb-4">
            <div class="card-body">
                <h2>Add New Item</h2>
                <form action="/item" method="POST">
                    <div class="mb-3">
                        <label for="name" class="form-label">Name:</label>
                        <input type="text" class="form-control" id="name" name="name" required>
                    </div>
                    <div class="mb-3">
                        <label for="description" class="form-label">Description:</label>
                        <textarea class="form-control" id="description" name="description" required></textarea>
                    </div>
                    <button type="submit" class="btn btn-primary">Add Item</button>
                </form>
            </div>
        </div>

        <!-- Item List -->
        <h2>Items</h2>
        <% if (items.length === 0) { %>
            <p>No items found.</p>
        <% } else { %>
            <div class="row">
                <% items.forEach((item, index) => { %>
                    <div class="col-md-4 mb-3">
                        <div class="card">
                            <div class="card-body">
                                <h5 class="card-title"><%= item.name %></h5>
                                <p class="card-text"><%= item.description %></p>
                                <div class="d-flex gap-2">
                                    <a href="/item/<%= index %>/edit" class="btn btn-warning">Edit</a>
                                    <form action="/item/<%= index %>/delete" method="POST" class="d-inline">
                                        <button type="submit" class="btn btn-danger">Delete</button>
                                    </form>
                                </div>
                            </div>
                        </div>
                    </div>
                <% }) %>
            </div>
        <% } %>
    </div>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

edit.ejs - Edit View

This view provides a form for editing existing items:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Edit Item - CRUD App</title>

                        <textarea class="form-control" id="description" name="description" required><%= item.description %></textarea>

                        <button type="submit" class="btn btn-primary">Update Item</button>
                        <a href="/" class="btn btn-secondary">Cancel</a>
                    </div>
                </form>
            </div>
        </div>
    </div>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>

View Features Explained

  • index.ejs:
    • Contains a form for creating new items at the top
    • Displays all items in a responsive grid layout
    • Each item card shows the name and description
    • Edit and Delete buttons for each item
    • Uses Bootstrap for styling and responsive design
  • edit.ejs:
    • Pre-filled form with the item's current data
    • Update and Cancel buttons
    • Consistent styling with the main view
    • Form submits to the update route with the item's ID

Note: The <%= %> syntax in EJS templates is used to output escaped HTML. This helps prevent XSS (Cross-Site Scripting) attacks by automatically escaping special characters.

Step 5: Implementing CRUD Operations

📚Learn: Routes tell your server which code to run when someone visits different URLs, like telling a receptionist which department should handle different types of calls.

Add these routes to your app.js:

// Create operation
app.post('/item', (req, res) => {
  const items = db.get('items') || [];
  items.push({
    name: req.body.name,
    description: req.body.description
  });
  db.set('items', items);
  res.redirect('/');
});

// Read operation (already implemented in the main route)

// Update operation - form
app.get('/item/:id/edit', (req, res) => {
  const items = db.get('items') || [];
  const item = items[req.params.id];
  res.render('edit', { item: item, id: req.params.id });
});

// Update operation - process
app.post('/item/:id/update', (req, res) => {
  const items = db.get('items') || [];
  items[req.params.id] = {
    name: req.body.name,
    description: req.body.description
  };
  db.set('items', items);
  res.redirect('/');
});

// Delete operation
app.post('/item/:id/delete', (req, res) => {
  const items = db.get('items') || [];
  items.splice(req.params.id, 1);
  db.set('items', items);
  res.redirect('/');
});

Step 6: Using GitHub Copilot

Setting up GitHub Copilot:

  1. Install GitHub Copilot extension in your code editor
  2. Sign in with your GitHub account
  3. Ensure your account has access to Copilot

Using Copilot for Development:

  • Prompt completions: Type a comment describing what you want to do, and Copilot will suggest code
  • Contextual awareness: Copilot understands the structure of your project
  • Learning from patterns: As you write more code, Copilot adapts to your style

Example Prompts for Copilot:

  • "// Create a function to validate form input"
  • "// Add a route to handle API requests"
  • "// Create a database query to filter items"

Step 7: Pushing to GitHub

  1. Initialize a Git repository: 
    git init
  2. Create a .gitignore file: 
    node_modules
data.json
  3. Add and commit your changes: 
    git add .
git commit -m "Initial commit - CRUD app setup"
  4. Create a repository on GitHub and follow the instructions to push your code

Running Your Application

  1. Start your application: 
    node app.js
  2. Open your browser and navigate to: http://localhost:3000
